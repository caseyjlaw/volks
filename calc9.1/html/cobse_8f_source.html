1  SUBROUTINE obsnt ( KOUNT, KEND)
2  IMPLICIT None
3 C
4 C 1. OBSNT
5 C
6 C 1.1 OBSNT PROGRAM SPECIFICATION
7 C
8 C 1.1.1 OBSNT calls the database handler to fetch an observation. If the
9 C end-of-data is detected, a flag is set and CALC finishes processing the
10 C experiment. Before each new observation is processed, OBSNT writes the
11 C observation number, time tag, the baseline identification, and the
12 C source name. OBSNT is called once for each observation.
13 C
14 C 1.2 OBSNT PROGRAM INTERFACE
15 C
16 C 1.2.1 CALLING SEQUENCE -
17 C INPUT VARIABLES:
18 C 1. KOUNT - THE VARIABLE WHICH INITIALIZES THE COUNTER OF THE
19 C OBSERVATION ITEMS TO ZERO IN SUBROUTINE INITL.
20 C OUTPUT VARIABLES:
21 C 1. KEND - THE VARIABLE WHICH FLAGS WHEN AN END OF FILE HAS
22 C BEEN REACHED. (NOTE: KEND = 0 MEANS PROCESS
23 C ANOTHER OBSERVATION ITEM, KEND = 1 MEANS THAT THE
24 C END OF FILE HAS BEEN REACHED.)
25 C
26 C 1.2.2 COMMON BLOCKS USED -
27  include 'ccon.i'
28 C
29 C 1.2.3 PROGRAM SPECIFICATIONS -
30  Integer*2 ITAG(5), KERR(5), LNBASE(4,2), LSTRNM(4), NDO(3), NN
31  Integer*4 KOUNT, KEND, N
32  Real*8 TAGSEC
33 C
34 C 1.2.4 DATA BASE ACCESS - CALL SUBROUTINE MVREC TO OBTAIN THE NEXT OBSERVATION
35 C FROM THE DATA BASE. CALL SUBROUTINE FINIS TO CLOSE
36 C THE DATA BASE.
37 C 'GET' VARIABLES:
38 C 1. ITAG(5) - AN ARRAY USED TO STORE THE YEAR, MONTH, DAY,
39 C HOUR, AND MINUTE OF THE OBSERVATION TIME TAG.
40 C 2. LNBASE(4,2) - AN ARRAY USED TO STORE THE EIGHT CHARACTER
41 C BASELINE IDENTIFICATIONS OF THE CURRENT
42 C OBSERVATION. (ALPHAMERIC)
43 C 3. LSTRNM(4) - AN ARRAY USED TO STORE THE EIGHT CHARACTER
44 C STAR NAME OF THE CURRENT OBSERVATION.
45 C (ALPHAMERIC)
46 C 4. TAGSEC - AN ARRAY USED TO STORE THE SECONDS PORTION OF
47 C THE OBSERVATION TIME TAG. (SEC)
48 C ACCESS CODES:
49 C 1. 'UTC TAG ' - THE DATA BASE ACCESS CODE FOR THE YEAR, MONTH,
50 C DAY, HOUR, AND MINUTE PORTION OF THE UTC
51 C OBSERVATION TIME TAG ARRAY. (Year is 2-digit
52 C in the Mark III analysis system.)
53 C l.5 'UTC TAG4' - THE DATA BASE ACCESS CODE FOR THE YEAR, MONTH,
54 C DAY, HOUR, AND MINUTE PORTION OF THE UTC
55 C OBSERVATION TIME TAG ARRAY. (New proposed
56 C L-code. Year will be 2-digits in the Mark
57 C III analysis system.)
58 C 2. 'BASELINE' - THE DATA BASE ACCESS CODE FOR THE BASELINE
59 C IDENTIFICATION ARRAY.
60 C 3. 'STAR ID ' - THE DATA BASE ACCESS CODE FOR THE SOURCE
61 C IDENTIFICATION ARRAY.
62 C 4. 'SEC TAG ' - THE DATA BASE ACCESS CODE FOR THE SECONDS
63 C PORTION OF THE UTC OBSERVATION TIME TAG.
64 C
65 C 1.2.5 EXTERNAL INPUT/OUTPUT -
66 C OUTPUT VARIABLES:
67 C 1. THE OBSERVATION NUMBER, TIME TAG, BASELINE IDENTIFICATION,
68 C AND SOURCE IDENTIFICATION. (NOTE: THESE VARIABLES ARE
69 C WRITTEN OUT ONCE FOR EACH OBSERVATION ITEM PROCESSED.)
70 C 2. POSSIBLE ERROR OUTPUT
71 C
72 C 1.2.6 SUBROUTINE INTERFACE -
73 C CALLER SUBROUTINES: MAIN
74 C CALLED SUBROUTINES: GETA, GETI, GET4, KILL, MVREC, FINIS
75 C
76 C 1.2.7 PROGRAM VARIABLES -
77 C 1. KEND - THE VARIABLE WHICH FLAGS WHEN AN END OF FILE HAS BEEN
78 C REACHED. (NOTE: KEND = 0 MEANS PROCESS ANOTHER
79 C OBSERVATION ITEM, KEND = 1 MEANS THAT THE END OF FILE
80 C HAS BEEN REACHED.)
81 C 2. KERR(5) - THE DATA BASE ERROR RETURN FLAGS.
82 C 3. NDO(3) -
83 C
84 C 1.2.9 PROGRAMMER - DALE MARKHAM 01/12/77
85 C PETER DENATALE 07/06/77

98 C
99 C OBSNT Program Structure
100 C
101 C Try to move to the next observation.
102  CALL mvrec ( 2, 1, 1, kerr(1) )
103 C
104 C Check the error return to determine course of action.
105 C KERR = 0 says observation found. KERR = 1 says end-of-file.
106 C First check for database failure.
107  IF ( kerr(1) .NE. 0 ) THEN
108  IF ( kerr(1) .EQ. 1 ) GO TO 810
109  CALL ckill (6hobsnt , 1, kerr(1) )
110  ENDIF
111 C
112 C GET the time tag, the baseline id, and source name.
113 C First try to get new time tag with 4-digit year. If not there, get old
114 C time tag with 2-digit year. Doesn't really matter though.
115  CALL geti ('UTC TAG4 ', itag, 5, 1, 1, ndo, kerr(2) )
116  If (kerr(2) .ne. 0)
117  * CALL geti ('UTC TAG ', itag, 5, 1, 1, ndo, kerr(2) )
118  CALL get4 ('SEC TAG ', tagsec, 1, 1, 1, ndo, kerr(3) )
119  CALL geta ('BASELINE ', lnbase, 4, 2, 1, ndo, kerr(4) )
120  CALL geta ('STAR ID ', lstrnm, 4, 1, 1, ndo, kerr(5) )
121 C
122 C Check for database errors and KILL if found.
123  DO n = 1,5
124  nn = n
125  IF ( kerr(n) .NE. 0 ) CALL ckill (6hobsnt ,nn,kerr(nn))
126  ENDDO
127 C
128 C Kick the counter for another observation and set end-of-data flag to zero.
129  kend = 0
130  kount = kount + 1
131 C
132 C Write the observation line to the screen.
133  IF (iluout .NE. -1)
134  .WRITE ( 6, 9000 ) kount, itag, tagsec, lnbase, lstrnm
135  9000 FORMAT (1x,i6,1x,i4,'/',i2,'/',i2,1x,i2,':',i2,f6.2,
136  1 1x,4a2,'-',4a2, 1x, 4a2 )
137  GO TO 820
138 C
139 C End-of-file encountered, so set KEND and close the database.
140  810 kend = 1
141  CALL finis(0)
142 C
143  820 RETURN
144  END
